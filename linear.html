Ⅱ. Sequential Structures

Ⅰ. 스택 (Stack)

정의: 스택은 한쪽 끝에서만 데이터의 삽입과 삭제가 일어나는 후입선출(LIFO) 방식의 선형 자료구조입니다.
장단점: 데이터의 추가와 제거가 $O(1)$의 속도로 매우 빠르지만, 중간 데이터에 직접 접근할 수 없습니다.
활용 사례: 웹 브라우저의 '뒤로 가기', 문서 편집기의 '되돌리기(Undo)' 기능 등에 활용됩니다.

Ⅱ. 큐 (Queue)

정의: 큐는 먼저 들어온 데이터가 먼저 나가는 선입선출(FIFO) 방식의 선형 자료구조입니다.
장단점: 데이터의 순서를 보장하며 처리하기에 적합하지만, 중간 요소 탐색이 비효율적입니다.
활용 사례: 프린터의 출력 대기열, 은행 창구의 번호표 시스템 등에 활용됩니다.

Ⅲ. 데크 (Deque)

정의: 데크는 양쪽 끝에서 삽입과 삭제가 모두 가능한 선형 자료구조입니다.
장단점: 스택과 큐의 특징을 모두 가져 매우 유연하지만, 구현이 상대적으로 복잡합니다.
활용 사례: 스케줄링 알고리즘, 주식 가격 변동 관리 등에 활용됩니다.
