Ⅲ. Tree Structures

Ⅰ. 이진 탐색 트리 (Binary Search Tree)

정의: 각 노드가 최대 두 개의 자식을 가지며, 왼쪽 자식은 부모보다 작은 값, 오른쪽 자식은 부모보다 큰 값을 가지도록 설계된 계층적 자료구조입니다.
장단점: 평균적으로 $O(\log n)$의 빠른 탐색과 삽입 성능을 보여주지만, 데이터가 순차적으로 입력되어 한쪽으로 치우친 편향 트리(Skewed Tree)가 될 경우 성능이 $O(n)$까지 저하될 수 있습니다.
사용 사례: 데이터베이스의 인덱싱 시스템이나 파일 시스템의 디렉토리 구조 탐색 등에 널리 활용됩니다.

Ⅱ. AVL 트리 (Self-Balancing Tree)

정의: 모든 노드에서 왼쪽 서브트리와 오른쪽 서브트리의 높이 차이가 1 이하가 되도록 스스로 균형을 유지하는 자가 균형 이진 탐색 트리입니다.
장단점: 트리가 항상 균형을 유지하므로 최악의 상황에서도 $O(\log n)$의 탐색 성능을 보장하지만, 삽입과 삭제 시마다 균형을 맞추기 위한 회전(Rotation) 연산이 추가되어 구현이 복잡합니다.
사용 사례: 삽입/삭제보다 검색 연산이 훨씬 빈번하게 일어나는 고성능 딕셔너리나 실시간 데이터 검색 엔진에 적합합니다.

Ⅲ. 최대 힙 (Max Heap)

정의: 부모 노드의 키값이 자식 노드의 키값보다 항상 크거나 같은 속성을 만족하는 완전 이진 트리 형식의 자료구조입니다.
장단점: 최댓값이나 최솟값을 O(1)의 시간 복잡도로 즉시 찾아낼 수 있어 매우 효율적이지만, 특정 데이터를 검색하는 용도로는 적합하지 않습니다.
사용 사례: 운영체제의 프로세스 스케줄링을 위한 우선순위 큐(Priority Queue) 구현이나 힙 정렬(Heap Sort) 알고리즘에 필수적으로 사용됩니다.
